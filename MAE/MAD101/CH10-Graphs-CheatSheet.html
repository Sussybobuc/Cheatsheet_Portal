📊 Graph Basics

Definition

G = (V, E) where V = vertices, E = edges

Directed Graph (Digraph)

Edges have direction: (u,v) ≠ (v,u)

Undirected Graph

Edges without direction: {u,v}

Degree

deg(v) = number of edges incident to v

In-degree: edges pointing to v

Out-degree: edges from v

Sum of degrees = 2|E| (each edge counted twice)

🎯 Graph Types

Simple Graph

No loops, no multiple edges between vertices

Multigraph

Multiple edges between same pair allowed

Pseudograph

Allows loops (edges to itself)

Complete Graph Kₙ

All vertices connected: |E| = n(n-1)/2

Cycle Cₙ

n vertices in a ring

Wheel Wₙ

Cycle Cₙ plus hub vertex

Bipartite Graph

Vertices split into two sets, edges only between sets

↔ Adjacency & Paths

Adjacent

Vertices u,v connected by edge

Path

Sequence v₁, v₂, ..., vₙ where consecutive vertices adjacent

Simple: no repeated vertices

Circuit (Cycle)

Path starting & ending at same vertex

Simple: no repeated vertices except endpoints

Length

Number of edges in path

Distance

Shortest path length between two vertices

u ↔ v: u reachable from v

🔗 Connectivity

Connected (Undirected)

Path exists between every pair of vertices

Connected Component

Maximal connected subgraph

Strongly Connected (Directed)

Path from u to v AND v to u for all pairs

Weakly Connected (Directed)

Connected if edges treated as undirected

Vertex Separation

Cut vertex: removal disconnects graph

Bridge: edge removal disconnects graph

Bipartite iff no odd cycles

🌳 Trees & Cycles

Tree

Connected acyclic graph

Properties: |E| = |V| - 1, unique path between any vertices

Forest

Acyclic graph (collection of trees)

Rooted Tree

One designated root vertex

Spanning Tree

Tree containing all vertices of G

Minimum Spanning Tree

Spanning tree with minimum total edge weight

Euler Path

Uses every edge exactly once

Exists iff graph connected & ≤2 vertices odd degree

✅ Graph Algorithms

BFS (Breadth-First Search)

Explore vertices by distance from start

Time: O(V + E)

DFS (Depth-First Search)

Explore vertices by depth, backtrack when needed

Time: O(V + E)

Dijkstra's Algorithm

Shortest path from source to all vertices

Requires non-negative weights

Time: O((V+E)log V) with heap

Kruskal's Algorithm

MST: sort edges, add if no cycle

Time: O(E log E)

Prim's Algorithm

MST: grow tree one edge at a time

Time: O(E log V) with heap

🗺️ Representations

Adjacency Matrix

n×n matrix M where M[i][j]=1 if edge (i,j)

Space: O(V²), time to check edge: O(1)

Adjacency List

For each vertex, list of adjacent vertices

Space: O(V+E), time to check edge: O(degree)

Edge List

List of all edges with weights (if any)

Space: O(E), good for sparse graphs

Incidence Matrix

(V × E) matrix; row=vertex, col=edge

Used for complex graph analysis

Sparse: E ≪ V² → adjacency list better

🔍 Special Properties

Planar Graph

Can draw on plane without edge crossings

Euler's formula: V - E + F = 2 (connected)

Colorability

Chromatic number: min colors to color vertices

Adjacent vertices must differ in color

Planar graphs: chromatic number ≤ 4

Isomorphic Graphs

Same structure, different labeling

Bijection f: V₁→V₂ preserving adjacency

Regular Graph

All vertices have same degree

Hamiltonian Path

Path visiting every vertex exactly once

NP-hard to determine if exists

⚙️ DFS Example

Algorithm

DFS(G, v):
  mark v as visited
  for each neighbor u of v:
    if u not visited:
      DFS(G, u)

Discovery Time / Finish Time

Useful for cycle detection & topological sort

Applications

  • Detect cycles in graphs
  • Topological ordering
  • Connected components
  • Strongly connected components

⚡ BFS Example

Algorithm

BFS(G, start):
  queue ← [start]
  mark start visited
  while queue not empty:
    v ← dequeue()
    for each neighbor u of v:
      if u not visited:
        mark visited
        enqueue(u)

Guaranteed Shortest Path

In unweighted graphs, finds shortest distance

Applications

  • Shortest path (unweighted)
  • Connected components
  • Bipartiteness checking
  • Level-by-level traversal

🌐 Graph Theory Theorems

Handshaking Lemma

Σ deg(v) = 2|E| (sum of degrees = twice edges)

Euler Path Theorem

Connected graph has Euler path iff exactly 0 or 2 odd vertices

Euler Circuit

Exists iff connected & all vertices even degree

Planar Graph Theorem

V - E + F = 2 for connected planar graphs

Corollary: E ≤ 3V - 6 for planar (V ≥ 3)

Four Color Theorem

Planar graph vertices colorable with ≤ 4 colors
