Induction Recursion Recurrence Verification

⟳ Mathematical Induction

Principle

To prove P(n) true ∀n ≥ base_case:

  1. Base case: Prove P(base) is true
  2. Inductive step: Assume P(k), prove P(k+1)

Inductive Hypothesis

The assumption P(k) made in step 2

∀n ∈ ℕ: P(n) is true

When to Use

Prove properties of natural numbers, sums, recursion

💪 Strong Induction

Principle

Assume ALL of P(base), P(base+1), ..., P(k), prove P(k+1)

Steps

  1. Base case(s): prove P(base), P(base+1), etc.
  2. Inductive step: assume P(i) ∀ base ≤ i ≤ k
  3. Prove P(k+1) from these

Stronger Hypothesis

Allows use of multiple previous cases

Sometimes easier than regular induction

Equivalence
Regular induction ≡ Strong induction (in power)

🔁 Recursion

Definition

Function defined in terms of itself

Must have base case(s) and recursive case(s)

Structure

function f(n):
  if n == base: return base_value
  else: return f(smaller_input)

Key Requirements

  • Base case: non-recursive termination
  • Progress: each call gets "closer" to base
  • Trust: assume recursive calls work
Recursive proof ≡ Induction proof

📝 Induction Examples

Sum Formula

Prove: ∑ᵏ₌₁ⁿ k = n(n+1)/2

Base: n=1: 1 = 1(2)/2 ✓

Step: Assume true for n=k

∑ᵏ₌₁ᵏ⁺¹ i = (∑ᵏ₌₁ᵏ i) + (k+1) = k(k+1)/2 + (k+1)

= (k+1)(k/2 + 1) = (k+1)(k+2)/2 ✓

Divisibility

Prove: 3 ∣ (4ⁿ − 1) for all n ≥ 0

Base: 4⁰ − 1 = 0, and 3 ∣ 0 ✓

Step: 4^(k+1) − 1 = 4·4ᵏ − 1 = 4(4ᵏ − 1) + 3

Both terms divisible by 3 ✓

ⁿ Recursive Examples

Factorial

factorial(n):
  if n = 0: return 1
  else: return n * factorial(n-1)

n! = n · (n-1)!

Fibonacci

fib(n):
  if n ≤ 1: return n
  else: return fib(n-1) + fib(n-2)

f(n) = f(n-1) + f(n-2)

⚠️ Exponential without memoization!

Binary Search

search(a, x, low, high):
  if low > high: return -1
  mid = (low+high)/2
  if a[mid] = x: return mid
  else if x < a[mid]: return search(..., low, mid-1)
  else: return search(..., mid+1, high)

⟲ Recurrence Relations

Definition

Equation expressing term via previous terms

aₙ = f(aₙ₋₁, aₙ₋₂, ..., aₙ₋ₖ)

Examples

Fibonacci: f(n) = f(n-1) + f(n-2), f(0)=0, f(1)=1

Merge Sort: T(n) = 2T(n/2) + O(n), T(1) = O(1)

Solving Methods

  • Iteration: expand to find pattern
  • Substitution: guess solution, verify
  • Master theorem: for divide-and-conquer
  • Characteristic equation: for linear recurrences
Linear recurrence: aₙ = c₁aₙ₋₁ + c₂aₙ₋₂ + ... + cₖaₙ₋ₖ

🔍 Solving Recurrences

Iteration Method

Expand recurrence, find pattern, verify

T(n) = T(n-1) + 1

T(n) = T(n-2) + 1 + 1 = T(n-3) + 3 = ... = T(1) + (n-1)

Result: T(n) = O(n)

Substitution (Guess & Check)

Guess solution form, verify by induction

For T(n) = 2T(n/2) + n: guess T(n) = O(n log n)

Check: T(n) = 2·cn/2·log(n/2) + n = cn·log n - cn + n

Telescoping

Rewrite so terms cancel

aₙ − aₙ₋₁ = f(n), sum from 1 to n

aₙ − a₀ = ∑ f(i)

💾 Memoization

Idea

Cache results of function calls; avoid recomputation

Example: Fibonacci

memo = {}
function fib(n):
  if n in memo: return memo[n]
  if n ≤ 1: result = n
  else: result = fib(n-1) + fib(n-2)
  memo[n] = result
  return result

Complexity Change

Without memo: O(2ⁿ) exponential

With memo: O(n) linear!

Trade-off

Time: improved | Space: O(n) for cache

🐛 Common Recursion Errors

Missing Base Case

Function never stops, infinite recursion

Wrong Base Case

Function returns incorrect value initially

No Progress

Recursive call doesn't get "smaller"

Example: f(n) = f(n) (infinite loop)

Stack Overflow

Too many recursive calls, stack exhausted

Solution: convert to iteration (tail recursion optimization)

Warning
Deep recursion can crash; use iteration for large inputs

🎯 Induction Proof Structure

Template

  1. Claim: State P(n) to prove
  2. Base Case: Prove P(0) or P(1)
  3. IH: "Assume P(k) for some k ≥ [base]"
  4. Step: Show P(k) ⟹ P(k+1)
  5. Conclusion: "By induction, P(n) ∀n"

Keys

  • Be explicit about assumptions
  • Clearly label base vs step
  • Show ALL work in step
  • Conclude with "QED" or similar

📊 Recursion Tree

Visualizing Recursion

T(n) = 2T(n/2) + n (Merge sort)

        n
       / \
     n/2 n/2    = n
     / \  / \
    /   \/   \   = n (4 calls)
   ...
   depth = log₂ n
   work per level = n
   total = n · log n

Counting Leaves

Leaves (base cases) = number of actual recursive returns

Internal nodes = remaining recursive calls

Work per Level

Sum of all nodes at depth d

Total = sum of work across all levels
