WAN Overview

What Is a WAN?

A Wide Area Network connects geographically dispersed LANs across large distances using service provider infrastructure. Unlike LANs (which you own), WAN links are typically leased from carriers (telcos, ISPs).

Common WAN Devices
DeviceRole
RouterDTE device; connects customer LAN to WAN link; Layer 3 forwarding
CSU/DSUChannel Service Unit / Data Service Unit; converts router serial signal to line signal; typically DCE device
ModemModulates/demodulates digital-to-analog signal; used for dial-up and DSL
WAN SwitchService provider equipment in the CO; switches packets/cells across provider network
ISDN TATerminal Adapter; connects non-ISDN devices to ISDN network
Layer 2 Protocols
WAN protocols operate at Layer 2 (Data Link). Common WAN Layer 2 protocols: HDLC, PPP, Frame Relay, ISDN, ATM.

DTE vs DCE

Definitions
TermFull NameRole
DTEData Terminal EquipmentCustomer side device (router); receives clock signal from DCE; connects to local loop
DCEData Circuit-terminating EquipmentService provider side (CSU/DSU); provides clock signal; sets line speed
DTE (Router) — serial cable — DCE (CSU/DSU) — WAN — DCE — DTE
Back-to-Back Serial (Lab Only)

When connecting two routers directly with a serial cable in a lab, one router acts as DCE and must provide the clock:

Router(config-if)# clock rate 64000
! Only required on the DCE side of back-to-back serial
! Use: show controllers serial 0/0 to verify DTE/DCE role
Identifying DTE vs DCE
show controllers serial 0/0 — if output says "DTE" the router expects clock from CSU/DSU. If "DCE", this end must set clock rate.

ISDN Channel Types
  • BRI = 2B+D = 2 × 64 Kbps bearer + 16 Kbps delta = 128 Kbps data
  • PRI (US) = 23B+D = 23 × 64 Kbps + 64 Kbps = 1.544 Mbps (T1)
  • PRI (EU) = 30B+D = 30 × 64 Kbps + 64 Kbps = 2.048 Mbps (E1)

Default Encapsulation
Cisco serial interfaces default to HDLC. Use show interfaces serial 0/0 to see encapsulation type. Change with encapsulation ppp (or frame-relay).

HDLC

High-Level Data Link Control
  • Cisco's default WAN encapsulation on serial interfaces
  • Cisco proprietary — Cisco added a proprietary Type field to standard HDLC to support multiple Layer 3 protocols; NOT interoperable with other vendors
  • Synchronous serial only; point-to-point
  • Lightweight — no authentication, no compression
Vendor Interoperability
Cisco HDLC cannot connect to a non-Cisco router. If connecting to another vendor, use PPP (open standard) instead.
Commands
encapsulation hdlc               ! restore default on serial interface
show interfaces serial 0/0       ! verify: "Encapsulation HDLC"
show interfaces serial 0/0       ! check line/protocol status
Verification Output
Serial0/0 is up, line protocol is up
  Encapsulation HDLC, loopback not set

PPP — Point-to-Point Protocol

Overview
  • Open standard — RFC 1661; works across all vendors
  • Supports multiple Layer 3 protocols (IP, IPX, AppleTalk) simultaneously via NCP
  • Supports authentication: PAP (clear-text) and CHAP (MD5 challenge)
  • Supports multilink (bundle multiple physical links), compression, error detection
PPP Sub-Protocols
Sub-ProtocolFull NameFunction
LCPLink Control ProtocolEstablish, configure, test, maintain, and terminate the PPP link. Negotiates: authentication, compression, multilink, error detection (magic number)
NCPNetwork Control ProtocolNegotiates upper-layer (Layer 3) protocols. IPCP for IP, IPXCP for IPX, etc. Multiple NCPs can run simultaneously.
PPP Session Establishment
  1. LCP Open — LCP negotiates link parameters (MRU, auth method, compression, magic number)
  2. Authentication — if configured: PAP (2-way handshake) or CHAP (3-way challenge/response)
  3. NCP Negotiation — IPCP negotiates IP addresses and compression for IP layer
  4. Link Up — data transfer begins; LCP monitors link health
Verification
show interfaces serial 0/0
! Look for: "Encapsulation PPP, LCP Open, multilink Open"
! IPCP Open = IP negotiated successfully
! Line protocol down = auth failed or LCP negotiation failed

PPP Configuration

Enable PPP Encapsulation
interface serial 0/0
  encapsulation ppp
Configure CHAP Authentication
! On BOTH routers (passwords must match):
username RouterB password cisco   ! remote router's hostname + shared secret
interface serial 0/0
  encapsulation ppp
  ppp authentication chap
Configure PAP Authentication
interface serial 0/0
  encapsulation ppp
  ppp authentication pap
  ppp pap sent-username RouterA password cisco
Optional PPP Features
ppp authentication chap pap      ! try CHAP first, fallback to PAP
ppp multilink                    ! enable multilink PPP (bundle links)
ppp compress predictor           ! enable compression
CHAP Username Rule
The username must match the hostname of the remote router exactly (case-sensitive). The password must be identical on both ends. CHAP uses MD5 hash — password never sent in clear text.

Exam Rule
CHAP is always preferred over PAP. Use CHAP unless the remote device does not support it.

Frame Relay & WAN Exam Rules

Frame Relay Key Concepts
  • Packet-switched — frames routed through shared provider network via virtual circuits
  • PVC (Permanent Virtual Circuit) — logically permanent; provisioned by provider; most common
  • SVC (Switched Virtual Circuit) — dynamically established; rare
  • DLCI (Data Link Connection Identifier) — 10-bit number identifying a VC; locally significant only
  • LMI (Local Management Interface) — signaling standard between router and Frame Relay switch; types: Cisco, ANSI, Q.933a
  • CIR (Committed Information Rate) — guaranteed bandwidth on a PVC
WAN Exam Key Rules
HDLC = Cisco default on serial; proprietary; no auth
PPP = open standard; use between different vendors
CHAP = MD5, 3-way handshake; preferred over PAP
PAP = clear text; 2-way; less secure
DCE provides clock; DTE receives clock
clock rate required on DCE side of back-to-back serial
Frame Relay = packet-switched; DLCI = local circuit ID
ATM = 53-byte fixed cells (5 header + 48 payload)
ISDN BRI = 2B+D = 128 Kbps user data
