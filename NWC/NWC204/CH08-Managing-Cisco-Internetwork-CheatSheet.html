Router Internal Components

ComponentContentsVolatile?Notes
ROMPOST, Bootstrap code, Mini-IOS (RxBoot / ROMMON)NoRuns first on boot; cannot be erased by user
FlashIOS image(s)NoErasable/programmable EEPROM; retains IOS across reboots; default IOS source
RAM (DRAM)Running config, routing table, ARP cache, packet buffers, IOS (running copy)YesLost on power-off or reload; running-config lives here
NVRAMStartup config, configuration registerNoSurvives reboots; startup-config loaded at boot
Quick Memory Rule
ROM=bootstrap | Flash=IOS | RAM=running | NVRAM=startup
Volatile vs Non-Volatile
RAM is volatile — running-config is lost on reboot unless saved to NVRAM with copy running-config startup-config.

Router Boot Sequence

  1. POST (Power-On Self Test) — hardware diagnostics; runs from ROM; verifies CPU, RAM, interfaces
  2. Load Bootstrap — small program in ROM; locates and loads IOS image
  3. Locate & Load IOS — searches: Flash (default) → TFTP server → ROM (mini-IOS); decompresses IOS into RAM
  4. Load Configuration — searches NVRAM for startup-config; if not found → Setup Mode (dialog); if TFTP configured → loads from network
Boot Location Order (IOS)
1. Flash → 2. TFTP Server → 3. ROM (mini-IOS)
Boot Location Order (Config)
1. NVRAM (startup-config) → 2. TFTP → 3. Setup Mode
Config Register Controls Boot
The configuration register (in NVRAM) tells the router where to find IOS and whether to load startup-config.

Configuration Register

Common Values
ValueModeDescription
0x2100ROMMONROM Monitor mode; manual boot; used for password recovery step 1
0x2101ROM BootBoot from mini-IOS in ROM (RxBoot)
0x2102Normal (default)Load IOS from Flash; load config from NVRAM; normal production setting
0x2142Ignore NVRAMBit 6 set; skip startup-config at boot; used for password recovery
Commands
show version                     ! shows current config-register value
config-register 0x2142           ! in global config mode
config-register 0x2102           ! restore to normal after password recovery
Where to Find It
show version displays the config register at the very bottom: "Configuration register is 0x2102"

Password Recovery Procedure

Use Case
Locked out of router — forgot enable password or enable secret. Requires physical console access.
0x2142 = ignore NVRAM → 0x2102 = normal boot
  1. Reboot & Send Break Signal — power-cycle or reload; within 60 seconds press Break (Ctrl+Break in HyperTerminal / Ctrl+C in SecureCRT) to enter ROMMON mode
  2. Change Config Register — at rommon 1> prompt type:
    confreg 0x2142 then reset (router reboots, skips NVRAM startup-config)
  3. Copy Startup to Running — after reboot, in privileged exec:
    copy startup-config running-config (loads old config into RAM without passwords enforced)
  4. Change the Passwords — set new enable secret:
    enable secret cisco
    Also change line passwords if needed: line con 0 → password cisco
  5. Restore Config Register & Save — config-register 0x2102, then copy running-config startup-config, then reload

Backing Up & Restoring IOS

Verification First
show flash
show flash:                      ! list IOS file(s), free space
Backup IOS to TFTP Server
copy flash: tftp:
! Router prompts for:
! Source filename [c2600-...bin]: (press Enter or type filename)
! Address or name of remote host: 192.168.1.100
! Destination filename [c2600-...bin]: (Enter)
Restore / Upgrade IOS from TFTP
copy tftp: flash:
! Router prompts for TFTP server IP and filename
! WARNING: will erase existing flash content (on some platforms)
Before Erasing Flash
Verify TFTP server is reachable (ping), file is on server, and you have sufficient flash space. Erasing flash without a valid IOS available = bricked router.
Verify Transfer
After copy, show flash to confirm new file is present and show version to see IOS version after reload.

Backing Up & Restoring Configs

Save & Backup Commands
copy running-config startup-config  ! save RAM config to NVRAM
copy startup-config running-config  ! load NVRAM config into RAM
copy running-config tftp:           ! backup running config to TFTP
copy tftp: running-config           ! merge TFTP config into running config
copy startup-config tftp:           ! backup startup config to TFTP
copy tftp: startup-config           ! overwrite startup config from TFTP
Erase / Reset
erase startup-config                ! wipe NVRAM (clears startup-config)
erase nvram:                        ! alternative to erase startup-config
reload                              ! reboot (goes to setup mode if NVRAM empty)
Short Forms
wr = write memory = copy run start (deprecated but still works on older IOS)
copy tftp: running-config Behavior
Merges (does not replace) the TFTP config into running-config. Conflicting commands may stack. Use with care in production.

CDP — Cisco Discovery Protocol

What Is CDP
  • Layer 2 protocol — Cisco proprietary; multicast to 01:00:0C:CC:CC:CC
  • Discovers directly connected Cisco devices only (not across routers)
  • Shares: device ID, IP addresses, platform, IOS version, port ID, capabilities
  • Runs on all Cisco routers, switches, and most Cisco devices by default
  • Does not use IP — works even if no IP is configured on interface
Timers
TimerDefaultDescription
CDP Timer60 secondsHow often CDP advertisements are sent
Hold Time180 secondsHow long to keep neighbor info before declaring it dead
Enable / Disable
no cdp run                          ! disable CDP globally
cdp run                             ! re-enable CDP globally
no cdp enable                       ! disable CDP on specific interface
cdp enable                          ! re-enable on interface

Telnet Management

Connecting to Remote Device
telnet 192.168.1.1              ! explicit telnet command
192.168.1.1                     ! shortcut - type IP directly at CLI
RouterA                         ! if hostname in host table or DNS
Session Management
Ctrl+Shift+6, then X            ! suspend current Telnet session
                                ! returns to LOCAL router without closing session
show sessions                   ! list all suspended Telnet sessions
resume                          ! resume most recent session
resume 2                        ! resume session number 2
disconnect 2                    ! close/terminate session number 2
show users                      ! show who is connected to THIS router via Telnet
Verifying Connectivity
ping 192.168.1.1                ! ICMP echo test
traceroute 192.168.1.1          ! trace path hop-by-hop
Ctrl+Shift+6 then X
This key sequence is critical for the exam. It suspends the Telnet session (keeps it alive) and returns you to the local router CLI. The session remains open and can be resumed.
show sessions vs show users
show sessions = Telnet sessions FROM this router to others.
show users = Telnet sessions TO this router from others.

Hostname Resolution

Method 1: Static Host Table
ip host RouterA 192.168.1.1
ip host RouterA 192.168.1.1 10.1.1.1  ! multiple IPs per host
show hosts                              ! display host table
Method 2: DNS
ip domain-lookup                       ! enable DNS (on by default)
ip name-server 192.168.1.100           ! specify DNS server IP
ip domain-name cisco.com               ! default domain suffix
! Now: ping RouterA resolves to RouterA.cisco.com via DNS
Disable DNS Lookup
By default, any mistyped command is sent as a DNS query, causing a long timeout. Disable with: 
no ip domain-lookup                    ! stops router DNS-resolving typos
                                       ! Essential in lab/study environments
Exam Tip
When you see a router freezing on a typo (trying to resolve as hostname), the fix is no ip domain-lookup. Press Ctrl+Shift+6 to interrupt the current lookup attempt.

CharMeaning

Traceroute

Debug Commands
debug ip icmp                   ! debug ICMP messages (use with care)
debug ip rip                    ! debug RIP updates
undebug all                     ! stop ALL debug output
no debug all                    ! same as undebug all
show processes                  ! CPU/process utilization
Debug Warning
Debug commands are CPU-intensive. debug all can crash a router under load. Always use undebug all when done. Prefer terminal monitor for remote debug output.
