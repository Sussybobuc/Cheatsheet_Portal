TCP — Transmission Control Protocol

Characteristics
  • Connection-oriented — 3-way handshake before data transfer
  • Reliable — acknowledgement (ACK) for every segment
  • Sequenced — sequence numbers guarantee correct ordering
  • Flow control — sliding window prevents buffer overflow
  • Error recovery — lost/corrupt segments retransmitted
  • Full-duplex — simultaneous bidirectional data flow
3-Way Handshake (Connection Establishment)
Client ──── SYN ──────────────► Server   (seq=x)
Client ◄─── SYN-ACK ─────────── Server   (seq=y, ack=x+1)
Client ──── ACK ──────────────► Server   (ack=y+1)
              [ Data Transfer Begins ]
4-Way Termination (Connection Teardown)
Client ──── FIN ──────────────► Server
Client ◄─── ACK ─────────────── Server
Client ◄─── FIN ─────────────── Server
Client ──── ACK ──────────────► Server   [ CLOSED ]

Windowing & Flow Control
Sliding Window
Sender can transmit up to window size bytes before requiring an ACK. Window grows on success (slow start), shrinks on congestion/loss. Prevents fast sender from overwhelming slow receiver.
Window=1 → send 1, wait for ACK → send 1, wait... Window=3 → send 3 segments, wait for ACK → send next 3

UDP — User Datagram Protocol

Characteristics
  • Connectionless — no handshake; just send and forget
  • Unreliable — no ACKs, no retransmission
  • Unsequenced — no ordering guarantees
  • Low overhead — 8-byte header (vs 20+ for TCP)
  • Fast — no connection setup delay
  • Upper-layer protocols must handle reliability if needed
UDP Header (only 4 fields, 8 bytes total)
┌──────────────┬──────────────┐
│  Source Port │   Dest Port  │  (16-bit each)
├──────────────┼──────────────┤
│    Length    │   Checksum   │  (16-bit each)
└──────────────┴──────────────┘
Protocols That Use UDP
  • DNS (port 53) — fast name lookups
  • TFTP (port 69) — simple file transfer
  • SNMP (port 161/162) — network management
  • DHCP (port 67/68) — uses broadcast addressing
  • NTP (port 123) — time synchronization
  • RIP (port 520) — routing updates
  • VoIP / Video streaming — latency critical, loss tolerated

DNS Uses Both TCP and UDP!
UDP port 53 for normal queries (<512 bytes). TCP port 53 for zone transfers or large responses (>512 bytes).
Exam Tip — FTP Ports
Port 21 = control (commands). Port 20 = data transfer (active mode only). Passive FTP uses ephemeral ports for data.

Cisco 3-Layer Hierarchical Model

Core Layer (Backbone)
  • Purpose: High-speed packet switching — the backbone
  • No packet manipulation, no access lists, no QoS
  • Maximum speed and reliability; redundant paths
  • Devices: high-end routers, multilayer switches (Cat 6500)
Distribution Layer (Workgroup)
  • Purpose: Policy-based connectivity and routing
  • Implements access lists, route filtering, summarization
  • VLAN routing, QoS, inter-VLAN communication
  • Boundary between core and access layers
  • Devices: routers, Layer 3 switches (Cat 3750)
Access Layer (Desktop)
  • Purpose: End-user device connectivity
  • Connects PCs, printers, IP phones, wireless APs
  • Port security, VLANs, PoE (Power over Ethernet)
  • Devices: Layer 2 switches (Cat 2960), hubs (legacy)
Design Rule
Core = speed only  |  Distribution = policy  |  Access = end users
Collapsed Core
Small networks merge Core + Distribution into one layer (collapsed core). One powerful switch handles both functions.

Collision & Broadcast Domains

Collision Domain

A segment where simultaneous transmissions cause a collision. Devices must use CSMA/CD. Reducing collision domains improves performance.

  • Hub: ALL ports = 1 collision domain
  • Switch: each port = separate collision domain
  • Router: each interface = separate collision domain
Broadcast Domain

All devices that receive a broadcast frame. Routers separate broadcast domains (do not forward broadcasts).

  • Hub/Switch (no VLANs): all ports = 1 broadcast domain
  • Router: each interface = separate broadcast domain
  • VLANs: each VLAN = separate broadcast domain
Worked Example
Topology: 1 Router, 2 interfaces Each interface → 1 switch, each switch → 12 hosts Collision domains: Router: 2 (one per interface) Switches: 24 (one per host port) Total: 26 collision domains Broadcast domains: Total: 2 (one per router interface)
Switches Don't Break Broadcast Domains
A switch by default forwards broadcasts to ALL ports. Only VLANs or routers separate broadcast domains.

CSMA/CD Process
  1. Device listens to check if the wire is idle (Carrier Sense)
  2. If idle, device begins transmitting frame (Multiple Access)
  3. Device continues listening while transmitting
  4. If collision detected: transmit 32-bit jam signal
  5. All devices in domain run random backoff timer
  6. After timer expires, attempt retransmission

MAC Addressing & Ethernet Frames

MAC Address Structure (48-bit / 6 bytes)
AA:BB:CC : DD:EE:FF ├── OUI ──┤├── Device ID ──┤ Organizationally Manufacturer-assigned Unique Identifier (unique per NIC)
  • Written as 12 hexadecimal digits (e.g., 00:1A:2B:3C:4D:5E)
  • OUI (first 3 bytes) = assigned to manufacturer by IEEE
  • Burned-In Address (BIA) — stored in ROM on NIC
  • Can be overridden in software (MAC spoofing)
  • Broadcast: FF:FF:FF:FF:FF:FF (all ones)
  • Multicast: first byte LSB = 1 (e.g., 01:00:5E:xx:xx:xx)

Data Encapsulation & De-encapsulation

Sender — Encapsulation (Application → Physical)
  1. Application (L7–L5): User creates data → Data
  2. Transport (L4): TCP/UDP header added with port numbers → Segment (TCP) / Datagram (UDP)
  3. Network (L3): IP header added with source & destination IP → Packet
  4. Data Link (L2): MAC header (src/dst MAC) + FCS trailer → Frame
  5. Physical (L1): Frame converted to electrical/optical/radio signals → Bits
Receiver — De-encapsulation (Physical → Application)
  1. Physical: Signals received, converted back to bits → frame
  2. Data Link: FCS checked (CRC), MAC header examined/stripped
  3. Network: IP header checked, routing decisions, header stripped
  4. Transport: TCP/UDP header processed, segments reassembled
  5. Application: Data delivered to requesting application
PDU Summary by Layer
L7-5  Application/Presentation/Session  → DATA
L4    Transport                          → SEGMENT (TCP) / DATAGRAM (UDP)
L3    Network                            → PACKET
L2    Data Link                          → FRAME
L1    Physical                           → BIT
What Each Header Contains
LayerPDUHeader Adds
Transport (L4)SegmentSrc Port, Dst Port, Seq#, Ack#, Flags, Window
Network (L3)PacketSrc IP, Dst IP, TTL, Protocol, Checksum
Data Link (L2)FrameSrc MAC, Dst MAC, Type/Length, FCS (trailer)
Important Distinction
IP addresses (L3) remain the same source-to-destination. MAC addresses (L2) change at every router hop — the router rewrites the frame with new MACs for the next segment.
TTL Field
Time-To-Live in IP header: each router decrements TTL by 1. When TTL=0, packet is discarded and ICMP "Time Exceeded" is sent to sender. Prevents infinite routing loops.
