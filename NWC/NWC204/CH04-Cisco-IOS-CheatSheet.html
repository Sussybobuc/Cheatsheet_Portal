Router Boot Process

  1. POST — Power-On Self Test verifies CPU, RAM, and interfaces via hardware diagnostics stored in ROM
  2. Bootstrap loads from ROM; searches for IOS image in Flash, TFTP, or ROM (in that order)
  3. IOS loads from Flash into RAM; decompresses if needed
  4. Startup-config copied from NVRAM into RAM, becoming the running-config
  5. If no startup-config found → router enters Setup Mode — answer 'no' to reach CLI directly
Configuration Register
0x2102 — default: boot IOS from Flash, load NVRAM config
0x2142 — skip NVRAM startup-config (password recovery)
0x2100 — always boot to ROMMON prompt
IOS Search Order
Flash → TFTP server → ROM (RxBoot)
Verify Boot
show version          ! IOS ver, register, uptime
show flash            ! IOS filename in Flash

Router Access Methods

Console Port — Out-of-Band Management
  • Physical: Rollover/Rolled cable (Cisco blue) + RJ-45 to DB-9 adapter to COM port
  • Software: HyperTerminal, PuTTY, SecureCRT, TeraTerm
  • Terminal settings: 9600 baud · 8 data bits · No parity · 1 stop bit · No flow control
  • Always accessible — even with no IP / IOS crash
  • Only one session at a time (line console 0)
Aux Port — Remote Out-of-Band
  • Designed for modem connection — dial-up remote management
  • Configured like console: line aux 0
Telnet — In-Band (Port 23)
  • Requires IP connectivity and VTY password
  • Sends all data unencrypted — avoid on untrusted networks
SSH — Secure In-Band (Port 22)
  • Encrypted — preferred over Telnet
  • Requires: hostname, domain name, RSA keys, local user account
SSH Quick Setup
ip domain-name lab.local
crypto key generate rsa modulus 1024
username admin secret cisco
line vty 0 4
 transport input ssh
 login local

Banner Types
! MOTD — shown BEFORE login (most seen)

WARNING: Unauthorized access is prohibited.

#

! Login banner — shown AT the login prompt
Router1(config)# banner login #

#

! EXEC banner — shown AFTER successful login
Router1(config)# banner exec #
Welcome to Router1. Changes are logged.
#
Delimiter Character
The # is the opening/closing delimiter. Use any character that does not appear in the message body (e.g., %, @, ^).
Banner Display Order
MOTD → Login → (auth) → EXEC

Password Configuration

Enable Passwords (Privileged EXEC)
! Legacy clear-text — shown in show run
Router(config)# enable password cisco

! MD5-hashed — always prefer this
Router(config)# enable secret class

! When BOTH are set: secret wins always
⚠ Secret vs Password Priority
enable secret always overrides enable password. If secret is set, password is ignored. Use enable secret exclusively.
Encrypt All Plaintext Passwords
Router(config)# service password-encryption
Encryption Types
service password-encryption → Type 7 (weak, reversible)
enable secret → Type 5 MD5 (strong, one-way)
Console Password (Line Console 0)
Router(config)# line console 0
Router(config-line)# password cisco
Router(config-line)# login
Router(config-line)# exec-timeout 5 0
Router(config-line)# logging synchronous
VTY Passwords (Telnet/SSH — Lines 0–4)
Router(config)# line vty 0 4
Router(config-line)# password cisco
Router(config-line)# login
Router(config-line)# exec-timeout 10 0
! For local auth (required for SSH):
Router(config-line)# login local

Aux Port Password
Router(config)# line aux 0
Router(config-line)# password cisco
Router(config-line)# login

line vty 0 4 = 5 simultaneous sessions. IOS 12.x+ supports 0 15 for 16 sessions.

Memory-to-Config Mapping
  • RAM holds running-config (active, lost on reload)
  • NVRAM holds startup-config (persistent)
  • Flash holds IOS image (persistent)

Configuration Management

View Configurations
show running-config        ! RAM — currently active
show startup-config        ! NVRAM — saved on disk
show running-config int f0/0  ! single interface only
Save Running-Config to NVRAM
! Preferred method:
copy running-config startup-config

! Shorthand (legacy, same result):
write memory
wr
Restore Startup-Config to RAM
copy startup-config running-config
! WARNING: This MERGES — does NOT replace.
! Reload for a clean restore.
Specific Interface Config
show running-config | section interface
show running-config | include ip address
Erase & Factory Reset
erase startup-config   ! wipe NVRAM config
write erase            ! same as above
delete nvram:          ! same as above
reload                 ! reboot after erase
TFTP Backup / Restore
! Save running-config to TFTP server:
copy running-config tftp:
  Address of remote host? 192.168.1.100
  Destination filename? router1-backup.cfg

! Restore from TFTP:
copy tftp: running-config
  Address of remote host? 192.168.1.100
  Source filename? router1-backup.cfg
⚠ Copy to Running-Config = Merge
Copying TO running-config only adds/overwrites — it does NOT remove existing commands. Use reload after copy startup running for a clean slate.

CDP — Cisco Discovery Protocol

CDP is a Cisco-proprietary Layer 2 protocol. It discovers directly connected Cisco devices — even with no IP configured — using multicast frames.

Key Fields Explained
  • IOS version — determines supported features
  • System image file — IOS filename in Flash
  • Uptime — time since last reload
  • Memory — total RAM / shared RAM

  • Flash (ATA) — IOS image storage
  • Config register — 0x2102 = normal boot

Password Recovery Procedure

  1. Connect to console, power cycle router
  2. Send Break (Ctrl+Break) within 60 seconds of boot → enter ROMMON
  3. At rommon> type: confreg 0x2142
  4. Type reset — router reboots, skips NVRAM
  5. Router boots without config; type no at setup dialog
  6. Enter enable (no password required)
  7. Load old config: copy startup-config running-config
  8. Change password: enable secret NewPass123
  9. Restore register: config-register 0x2102
  10. copy running-config startup-config then reload
⚠ Physical Access Required
Password recovery requires console cable and physical presence at the device.
