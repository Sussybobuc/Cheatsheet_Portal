Why Subnet? — Concepts & Benefits

Reasons to Subnet
  • Reduce broadcast domains — fewer hosts hear each broadcast; better performance
  • Efficient address utilization — allocate only what's needed per segment
  • Improved security — routers between subnets apply ACLs to control traffic
  • Easier management — logical grouping by department, floor, function
  • Fault isolation — problems contained to one subnet
  • Support for VLANs — each VLAN needs its own subnet
Subnet Mask Basics

A subnet mask is a 32-bit value with consecutive 1s on the left (network) and 0s on the right (host). Written in dotted decimal or CIDR prefix notation.

IP Address: 192.168.1.130 = 11000000.10101000.00000001.10000010 Subnet Mask: 255.255.255.128 = 11111111.11111111.11111111.10000000 Network: 192.168.1.128 (AND result) Host portion: .00000010 = host 2 in this subnet

How Subnetting Works

Borrow bits from the host portion and add them to the network portion. Each borrowed bit doubles the number of subnets but halves the hosts per subnet.

Class C /24:  NNNNNNNN.NNNNNNNN.NNNNNNNN.HHHHHHHH
Borrow 2:     NNNNNNNN.NNNNNNNN.NNNNNNNN.NNHHHHHH  (/26)
              n=2 borrowed → 2^2=4 subnets, 2^6-2=62 hosts

Key Subnetting Formulas

Core Formulas
Number of Subnets = 2^n n = number of BORROWED bits (subnet bits) Hosts per Subnet = 2^h - 2 h = number of HOST bits remaining (-2 for network address and broadcast address) Block Size (Increment) = 256 - interesting octet value e.g., mask=255.255.255.192 → 256-192=64 block size

Fast Subnetting Method (Block Size)

The Block Size Trick

The block size is the increment between subnets. Find it with: 256 − mask_value

Mask = 255.255.255.192 Block Size = 256 - 192 = 64 Subnets start at: 0, 64, 128, 192 (next subnet starts at previous + block size)
Step-by-Step Fast Method
  1. Identify the mask and find the interesting octet (the non-255, non-0 octet)
  2. Compute block size: 256 - interesting_octet
  3. List subnets: start at 0, keep adding block size until you exceed 255
  4. For each subnet: Network=first address, Broadcast=next_subnet-1, Hosts=between
Example: 192.168.10.0 with /27 (mask=.224)
Block size = 256 - 224 = 32 Subnet 0: Network=192.168.10.0, Hosts=.1-.30, BC=.31 Subnet 1: Network=192.168.10.32, Hosts=.33-.62, BC=.63 Subnet 2: Network=192.168.10.64, Hosts=.65-.94, BC=.95 Subnet 3: Network=192.168.10.96, Hosts=.97-.126, BC=.127 Subnet 4: Network=192.168.10.128, Hosts=.129-.158,BC=.159 Subnet 5: Network=192.168.10.160, Hosts=.161-.190,BC=.191 Subnet 6: Network=192.168.10.192, Hosts=.193-.222,BC=.223 Subnet 7: Network=192.168.10.224, Hosts=.225-.254,BC=.255
Find Which Subnet an IP Belongs To
  1. Find block size (256 - mask)
  2. Divide host octet by block size (integer division)
  3. Multiply result by block size = subnet network address
IP: 192.168.10.73 Mask: /27 (block=32) 73 / 32 = 2 (integer) 2 x 32 = 64 → Subnet = 192.168.10.64 Broadcast = 192.168.10.95 (64+32-1)
Quick IP Address Check
To determine if two IPs are on the same subnet: AND both IPs with the subnet mask. If results match, they are on the same subnet.

Why /30 is the Smallest Usable
/30 = 2 host bits → 2^2-2 = 2 usable hosts. One for each router interface on a P2P link. /31 and /32 have 0 usable hosts (2^1-2=0, 2^0-2=-1). RFC 3021 allows /31 for P2P but rarely used on CCNA.

VLSM — Variable Length Subnet Masks

VLSM allows allocating subnets of different sizes from the same address space. Maximizes address efficiency. Requires classless routing protocol (OSPF, EIGRP, RIPv2, BGP — NOT RIPv1 or IGRP).

VLSM Design Rule
Assign largest subnets FIRST, then progressively smaller ones. Work from largest requirement down to smallest. Ensure no subnet ranges overlap!
Worked Example — 192.168.10.0/24 — Requirements: 60, 30, 12, 2 hosts
Step 1 — Allocate 60-host subnet
Need 60 hosts → 2^h - 2 ≥ 60 h=6 → 2^6-2=62 hosts ✓ → /26 Subnet: 192.168.10.0/26 Network: 192.168.10.0 Hosts: 192.168.10.1 – .62 Broadcast: 192.168.10.63
Step 2 — Allocate 30-host subnet
Need 30 hosts → 2^h - 2 ≥ 30 h=5 → 2^5-2=30 hosts ✓ → /27 Start after previous: .64 Subnet: 192.168.10.64/27 Network: 192.168.10.64 Hosts: 192.168.10.65 – .94 Broadcast: 192.168.10.95
Step 3 — Allocate 12-host subnet
Need 12 hosts → 2^h - 2 ≥ 12 h=4 → 2^4-2=14 hosts ✓ → /28 Start after previous: .96 Subnet: 192.168.10.96/28 Network: 192.168.10.96 Hosts: 192.168.10.97 – .110 Broadcast: 192.168.10.111
Step 4 — Allocate 2-host WAN link
Need 2 hosts → 2^h - 2 ≥ 2 h=2 → 2^2-2=2 hosts ✓ → /30 Start after previous: .112 Subnet: 192.168.10.112/30 Network: 192.168.10.112 Hosts: 192.168.10.113 – .114 Broadcast: 192.168.10.115

No Overlapping Subnets!
Each IP address must belong to exactly one subnet. Overlapping subnets cause routing failures. Always verify ranges don't intersect when designing VLSM.

CIDR & Route Summarization

CIDR (Classless Inter-Domain Routing) allows route summarization (aggregation): combining multiple contiguous networks into one summary route entry to reduce routing table size.

Supernetting / Route Aggregation

Combine multiple subnets into one summary route. The summary covers the entire range — routers only need ONE entry instead of many.

Example — Summarize 4 Class C networks
Networks to summarize: 192.168.0.0/24 192.168.1.0/24 192.168.2.0/24 192.168.3.0/24 In binary (3rd octet): 0 = 00000000 1 = 00000001 2 = 00000010 3 = 00000011 Common prefix in 3rd octet: 000000xx (22 bits match) Summary route: 192.168.0.0/22 Covers: 192.168.0.0 – 192.168.3.255
Finding the Summary Route — Steps
  1. Write all network addresses in binary
  2. Find the common bit prefix (where they first differ)
  3. Count matching bits from left = new prefix length
  4. Zero out non-matching bits = summary network address
Route Summarization Benefits
  • Smaller routing tables → faster lookups
  • Less routing protocol traffic (fewer updates)
  • Hides topology changes behind the summary
  • Masks internal subnet failures from upstream routers

Troubleshooting IP Addressing

Common IP Configuration Problems
ProblemSymptomHow to IdentifyFix
Wrong subnet mask Cannot reach some hosts; reaches others incorrectly ipconfig/ifconfig; compare mask to others on segment Correct the subnet mask to match the network design
Wrong default gateway Local hosts reachable; remote networks unreachable ping gateway; tracert shows failure at first hop Set gateway to router interface IP on same subnet
Duplicate IP address Intermittent connectivity; ARP conflicts ARP cache shows same IP twice; OS warns of duplicate Identify conflicting host; change one IP to unique
IP in wrong subnet Cannot communicate with any local hosts Network address doesn't match after ANDing with mask Assign IP from correct subnet range
Overlapping subnets Routing loops; intermittent packet loss Check routing table for overlapping prefixes Redesign addressing plan; ensure no overlap
DHCP exhausted pool New devices get 169.254.x.x (APIPA) DHCP server logs; ipconfig shows APIPA address Expand DHCP pool or reduce lease time
Host using broadcast/network addr Host unreachable; may cause issues for whole subnet Check if IP ends in .0 or .255 (or subnet boundary) Assign valid host IP (not first or last in range)
Troubleshooting Process
  1. ping 127.0.0.1 — tests local TCP/IP stack (if fails: reinstall/repair TCP/IP)
  2. ping own IP — tests NIC and basic configuration
  3. ping default gateway — tests local subnet connectivity
  4. ping remote IP — tests routing; if fails check routing table and gateway
  5. ping by hostname — tests DNS resolution (if fails, check DNS server setting)
Cisco IOS Verification Commands
show ip interface brief        ! Summary of all interfaces (IP + status)
show ip route                  ! Routing table
show interfaces                ! Detailed interface stats
show arp                       ! ARP cache (IP to MAC mappings)
ping 192.168.1.1               ! Test reachability
traceroute 192.168.1.1         ! Trace hop-by-hop path
debug ip packet                ! Packet-level debugging (use carefully!)

Decision Chart — How Many Bits to Borrow?
Need X subnets? → borrow n bits where 2^n >= X Need Y hosts? → keep h bits where 2^h - 2 >= Y Prefix = class_default + n Verify: n + h = remaining host bits in class (Class C: n+h=8; Class B: n+h=16; Class A: n+h=24)

IP Addressing Design Tips & Exam Review

Key Exam Rules to Remember
  • First address in subnet = Network address (all host bits = 0) — NOT assignable
  • Last address in subnet = Broadcast address (all host bits = 1) — NOT assignable
  • Valid host range is everything between these two
  • 2^h - 2 usable hosts (subtract 2 always for network + broadcast)
  • 2^n subnets (n = borrowed bits, no subtraction needed in modern Cisco)
  • Smallest subnet for WAN = /30 (2 hosts per subnet)
