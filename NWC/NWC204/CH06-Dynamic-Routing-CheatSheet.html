Distance-Vector Key Characteristics
  • Sends full routing table to directly connected neighbors
  • Uses periodic updates on a timer (30s for RIP)
  • No topology map — only knows distance + direction
  • "Routing by rumor" — trusts neighbor information
  • Prone to routing loops without loop prevention
  • Slower convergence than link-state
Link-State Key Characteristics
  • Floods LSAs to ALL routers in the area
  • Each router builds identical topology database
  • Runs Dijkstra SPF independently
  • Triggered updates — only when topology changes
  • Fast convergence, scalable with areas
  • More CPU/memory intensive

RIP Configuration

RIP uses hop count as its only metric. Maximum 15 hops — hop count 16 = unreachable/infinity.

Enable RIP (Version 1 — Classful)
Router(config)# router rip
Router(config-router)# network 172.16.0.0
Router(config-router)# network 192.168.1.0
! Use classful network addresses only (RIPv1)
Enable RIP Version 2 (Classless)
Router(config)# router rip
Router(config-router)# version 2
Router(config-router)# network 172.16.0.0
Router(config-router)# network 192.168.1.0
! Optional — disable auto-summarization:
Router(config-router)# no auto-summary
Passive Interface (stop sending updates)
Router(config-router)# passive-interface f0/0
! Receives RIP on f0/0 but does NOT send updates
! Use on stub LAN segments (no other routers)
Verification
show ip route             ! R entries = RIP routes
show ip protocols         ! RIP config, networks, timers
debug ip rip              ! live RIP update messages
undebug all               ! stop all debug output
network Command
The network command in RIP uses classful network addresses — it enables RIP on ALL interfaces that fall within that classful range.

RIP Timers

TimerDefaultPurpose
Update30 secFrequency of full routing table broadcasts to neighbors
Invalid180 secRoute marked invalid if no update received for 180s (6 updates missed)
Holddown180 secIgnores new info about failed route — loop prevention
Flush240 secRoute removed from table after 240s of no updates
Timer Sequence for Failed Route
0s: Update missed → 180s: Invalid → Holddown starts → 240s: Flush → Route removed
View and Change Timers
show ip protocols           ! shows current timers
! Change timers (must match all routers!):
Router(config-router)# timers basic 30 180 180 240
⚠ Timer Mismatch
All RIP routers in the domain must have identical timer values. Mismatched timers cause unpredictable behavior and routing instability.
RIP Metric Limit
  • Maximum metric: 15 hops
  • 16 hops = infinity (network unreachable)
  • Limits RIP to small networks (max 15 router hops end-to-end)

IGRP Configuration

IGRP is Cisco proprietary, classful, distance-vector. Uses a composite metric based on bandwidth and delay (primarily).

Configure IGRP
! AS number must match all routers in domain
Router(config)# router igrp 100
Router(config-router)# network 172.16.0.0
Router(config-router)# network 192.168.1.0

! Passive interface:
Router(config-router)# passive-interface f0/0
IGRP Verification
show ip route               ! I entries = IGRP routes
show ip protocols           ! AS#, networks, timers
debug ip igrp events        ! summary IGRP activity
debug ip igrp transactions  ! detailed IGRP updates
undebug all                 ! stop debug output
IGRP Hop Count Limit
  • Default maximum: 100 hops
  • Configurable up to 255 hops
  • Much larger than RIP's 15-hop limit
Router(config-router)# metric maximum-hops 255
IGRP AS Number
The Autonomous System number in router igrp 100 must be the same on all routers to exchange routes. Different AS = no route exchange.
IGRP in Modern Networks
IGRP is deprecated and not supported in IOS 12.3(14)T and later. EIGRP supersedes it. IGRP appears on legacy CCNA exams for comparison.

IGRP Composite Metric

IGRP Metric Formula
Metric = [K1×BW + (K2×BW)/(256-load) + K3×delay] × [K5/(reliability+K4)]
Default K-Values
K-ValueComponentDefaultActive?
K1Bandwidth1Yes
K2Load0No
K3Delay1Yes
K4Reliability0No
K5Reliability0No
Simplified (default K1=1, K3=1):
Metric = Bandwidth + Delay
IGRP Bandwidth Calculation
  • BW = 10,000,000 / interface bandwidth (Kbps)
  • Serial 64K → BW component = 10,000,000/64 = 156,250
  • Ethernet 10M → BW component = 10,000,000/10,000 = 1,000

Loop Prevention Mechanisms

1. Maximum Hop Count
RIP: 16 hops = infinity (unreachable)
IGRP: 256 hops = infinity (unreachable)

Packets looping indefinitely will eventually exceed the maximum metric and be discarded, breaking the loop.

2. Split Horizon
Never advertise a route back out the interface from which it was learned.

Split horizon prevents two-node routing loops. If Router A learned a route from Router B, there's no point advertising it back — B already knows.

! Split horizon is ON by default on serial links
! Disable (frame relay hub-and-spoke):
Router(config-if)# no ip split-horizon
3. Route Poisoning (Poison Reverse)
When a route fails, immediately advertise it with metric = infinity (16 for RIP).

Instead of simply removing a failed route, the router poisons it by sending an update with metric=16. This tells neighbors immediately that the network is unreachable, without waiting for timeouts.

4. Holddown Timer
After a route fails, ignore any new information about that route for the holddown period.

When a route goes bad, the holddown timer starts. During holddown, the router rejects updates that claim to reach the failed network at a worse metric. This prevents an already-looping bad route from being re-accepted.

  • RIP Holddown: 180 seconds
  • IGRP Holddown: 280 seconds
  • Router will accept an update with BETTER metric during holddown
5. Triggered Updates (Flash Updates)
Send an immediate update when a route change occurs — don't wait for the periodic timer.

Triggered updates speed up convergence. Instead of waiting 30 seconds for the next RIP update, the router immediately notifies neighbors of the topology change.

AD Recap + RIP vs IGRP

Simultaneous RIP + IGRP

If both RIP and IGRP are running and both know a route, the router uses Administrative Distance to choose:

ProtocolADWins?
IGRP100✓ Yes — lower AD
RIP120✗ No — higher AD
AD Tiebreaker
If IGRP and RIP both have routes to 10.0.0.0/8, the IGRP route (AD=100) is installed in the routing table. The RIP route is kept as backup but not used.
RIP + IGRP Config Example
router rip
 version 2
 network 192.168.1.0
 network 172.16.0.0

router igrp 100
 network 192.168.1.0
 network 172.16.0.0
Redistribution
By default RIP and IGRP do NOT share routes with each other. Routes must be redistributed between protocols to share across protocol boundaries.
Manually Set Admin Distance
Router(config-router)# distance 95
! Changes AD for this routing protocol instance
! Use carefully — changes which routes are preferred

Troubleshooting Checklist
  • Both routers have matching RIP version (1 or 2)
  • IGRP AS numbers match on all routers
  • network statement covers the interface network
  • Interfaces are up/up (not shutdown)
  • No ACL blocking UDP port 520 (RIP)
  • Passive interfaces not blocking needed updates
  • Check no auto-summary if using VLSM with RIPv2
