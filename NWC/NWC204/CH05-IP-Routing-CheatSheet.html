Routing Fundamentals

A router forwards packets by examining the destination IP address and matching it to the best entry in its routing table.

Routing Table Entry Components
FieldDescriptionExample
NetworkDestination network address172.16.1.0
MaskSubnet mask / prefix length/24 or 255.255.255.0
Next-hopIP of next router toward destination172.16.2.1
Exit InterfaceLocal interface to send packet outSerial0/0
ADAdministrative distance (source trust)110 for OSPF
MetricCost to reach destinationhop count, bandwidth
How Routes are Added
  • Connected (C) — automatically when interface is up with IP
  • Static (S) — manually configured with ip route
  • Dynamic (R/O/I/D) — learned via routing protocols
Best route = longest prefix match → lowest AD → lowest metric

Administrative Distance (AD)

Administrative Distance determines which route source is preferred when multiple protocols know a route. Lower = more trusted.

IP Routing Process — Step-by-Step Packet Walk

  1. Source host creates IP packet with destination IP address of remote host
  2. Host checks: is destination on same subnet? If yes, ARP for MAC and send directly. If no, send to default gateway
  3. Host ARPs for the default gateway MAC address, encapsulates packet in an Ethernet frame addressed to the gateway MAC
  4. Router receives frame on ingress interface, checks destination MAC — matches its own MAC, accepts the frame
  5. Router strips (de-encapsulates) the Layer 2 frame, exposing the Layer 3 IP packet
  6. Router reads the destination IP address from the IP header
  1. Router performs routing table lookup — finds longest prefix match for destination IP
  2. Router decrements TTL by 1. If TTL = 0, discard and send ICMP Time Exceeded
  3. Router determines: exit interface + next-hop IP address
  4. If next-hop is Ethernet: ARP for next-hop MAC (or use ARP cache)
  5. Router re-encapsulates packet in NEW Layer 2 frame with: new source MAC (router's egress int MAC) and new dest MAC (next-hop MAC)
  6. Frame transmitted out the correct egress interface toward destination
Key Rule — L2 Changes, L3 Does Not
At every router hop, the Layer 2 frame is rebuilt (new source/dest MAC). The Layer 3 IP header stays the same (source and destination IP never change, except with NAT).

Route Selection Logic

Selection Priority (in order)
  1. Longest prefix match — most specific route wins regardless of source
  2. Lowest Administrative Distance — if same prefix from multiple sources
  3. Lowest Metric — if same AD (multiple paths from same protocol)
  4. Equal-cost load balancing — if metric also equal
Longest Prefix Example
Packet to 10.1.1.5 — routing table has:
10.0.0.0/8 via 1.1.1.1
10.1.1.0/24 via 2.2.2.2 ← WINS (more specific)
No Match = Packet Dropped
Packet Dropped
If no route matches and no default route exists, router drops the packet and sends ICMP Destination Unreachable to source.
Gateway of Last Resort

The ip route 0.0.0.0 0.0.0.0 default route matches ALL destinations not in the table — the "catch-all" route.

0.0.0.0/0 — matches every possible destination

Static Route Configuration

Syntax
ip route [dest-network] [mask] [next-hop-IP | exit-interface] [AD]
Static Route Examples
! Via next-hop IP address (preferred for point-to-point)
Router(config)# ip route 172.16.1.0 255.255.255.0 172.16.2.1

! Via exit interface (Ethernet — triggers proxy ARP)
Router(config)# ip route 192.168.3.0 255.255.255.0 Serial0/0

! Multiple static routes
Router(config)# ip route 10.1.0.0 255.255.0.0 10.0.0.2
Router(config)# ip route 10.2.0.0 255.255.0.0 10.0.0.2
Router(config)# ip route 10.3.0.0 255.255.0.0 10.0.0.2

! Floating static (backup, AD=130 beats out nothing until OSPF fails)
Router(config)# ip route 10.0.0.0 255.0.0.0 1.1.1.2 130

! Remove a static route
Router(config)# no ip route 172.16.1.0 255.255.255.0 172.16.2.1
Next-Hop vs Exit Interface
Use next-hop IP on multi-access (Ethernet) — router knows which neighbor to ARP for.
Use exit interface on point-to-point (Serial) — only one device on the link.

Default Route

The default route is a catch-all — it matches ANY destination not explicitly listed in the routing table.

Configure Default Route
! Quad-zero route — matches everything
Router(config)# ip route 0.0.0.0 0.0.0.0 [next-hop]

! Examples:
Router(config)# ip route 0.0.0.0 0.0.0.0 172.16.1.1
Router(config)# ip route 0.0.0.0 0.0.0.0 Serial0/0
Propagate to Other Routers
! In RIP — advertise default to neighbors:
Router(config-router)# default-information originate

! In OSPF:
Router(config-router)# default-information originate
Verify Default Route
show ip route
! Look for: Gateway of last resort is X.X.X.X
! Look for: S* 0.0.0.0/0 [1/0] via X.X.X.X
S* Notation
S* = Static + Candidate Default. The asterisk (*) marks the active default route (gateway of last resort).
Stub Router Use Case
A branch router with only one path to the network — configure a default route pointing to HQ instead of full routing table.

Bidirectional Routes Required
Every router needs routes in both directions. Missing a return route causes one-way reachability (packets go but replies can't return).

How traceroute Works
  • Sends UDP datagrams with incrementing TTL (starts at 1)
  • Each router decrements TTL — when TTL=0, sends ICMP Time Exceeded back
  • Source records that router's IP and latency
  • TTL increments each round until destination reached

Diagnostic Sequence
  1. show ip interface brief — are interfaces up/up?
  2. show ip route — is destination in table?
  3. ping [next-hop] — can we reach adjacent router?
  4. traceroute [dest] — where does it fail?
  5. Check remote router — does it have a return route?
