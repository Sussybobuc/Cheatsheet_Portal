Hub Era Problems

Single Collision Domain
All devices share the same physical medium. Only one can transmit at a time; all others must wait (CSMA/CD). Any collision affects every device.
Problems with Hubs
  • Shared bandwidth: all ports compete for the same total throughput
  • Half-duplex only: cannot send and receive simultaneously
  • No intelligence: floods every frame out all ports — always
  • One broadcast domain: every host receives every broadcast
  • Scalability: more hosts = more collisions = worse performance
Switching Solution
Each switch port = its own collision domain. Full-duplex capable. Frames forwarded only to the correct port. Bandwidth per port, not shared.

Three Switching Functions

  1. Address Learning — reads the source MAC of every incoming frame; maps it to the ingress port; stores the pair in the CAM table dynamically
  2. Forward / Filter Decisions — if destination MAC is known → forward only to that port (filtering); if unknown → flood all ports except source; if source port = dest port → filter (drop)
  3. Loop Avoidance — runs STP (IEEE 802.1D) to detect redundant paths and block ports that would cause switching loops; prevents broadcast storms and MAC table instability
Key Distinction
Layer 2 switches forward based on MAC addresses. Routers forward based on IP addresses (Layer 3). Switches do NOT break up broadcast domains without VLANs.

Bottom Line
Bridges and switches use identical logic (MAC learning + STP). Switches are simply faster, hardware-accelerated multi-port bridges.

MAC Address Table (CAM Table)

How the CAM Table Is Built
  1. Frame arrives on Port Fa0/1
  2. Switch reads source MAC from frame header
  3. Stores AA:BB:CC:DD:EE:FF → Fa0/1 in CAM table
  4. Looks up destination MAC in CAM table
  5. Known dest → forward to that port only; Unknown → flood all except source
Key Facts
  • CAM = Content Addressable Memory — hardware hash table for fast lookup
  • Age-out timer: 300 seconds — unused entries are removed
  • Unknown unicast: destination not in table → flooded (until learned)
  • Broadcasts & multicasts: always flooded to all ports in VLAN
  • Filtering: known destination on same port as source → frame dropped

STP Overview (IEEE 802.1D)

Problem: Switching Loops
Redundant physical links without STP cause: broadcast storms (frames loop endlessly), MAC table instability (same MAC seen on multiple ports), and multiple frame delivery. The network melts down in seconds.
STP Solution

Logically blocks redundant ports to create a loop-free Layer 2 topology while keeping physical redundancy for failover.

Bridge ID = 2-byte Priority + 6-byte MAC Address (8 bytes total)
STP Election Steps
  1. Elect a Root Bridge — switch with lowest Bridge ID
  2. Each non-root switch selects a Root Port — lowest-cost path to root
  3. Each segment selects a Designated Port — one forwarding port per link
  4. All remaining ports become Non-Designated (Blocked)
Root Bridge Election
Lowest Bridge ID wins. Default priority = 32768. Lower priority number = preferred root. Tie-break: lowest MAC address. To force root: lower the priority (spanning-tree vlan 1 priority 4096).

Total STP Convergence = 20s (Max Age / Blocking) + 15s (Listening) + 15s (Learning) = ~50 seconds
STP Timers (set on Root Bridge, propagated via BPDUs)
  • Hello Timer: 2 sec — frequency of BPDU transmission by root bridge
  • Max Age: 20 sec — time a non-root switch waits in Blocking before assuming root failure
  • Forward Delay: 15 sec — time spent in each of the Listening and Learning states

RSTP (IEEE 802.1w)

Rapid Spanning Tree Protocol
Reduces convergence from ~50 seconds to 1–2 seconds. Backward compatible with 802.1D.

RSTP Port States
  • Discarding — replaces Blocking + Listening + Disabled
  • Learning — same as 802.1D
  • Forwarding — same as 802.1D
RSTP Port Roles
  • Root: best path to root bridge
  • Designated: forwarding port on each segment
  • Alternate: backup path to root (was blocked non-designated)
  • Backup: redundant designated port on same segment

Exam Tip
Cut-Through = lowest latency, no error checking. Store-and-Forward = highest latency, full CRC verification. Fragment-Free = middle ground; 64 bytes = collision slot time; filters runts only. Modern Cisco Catalyst switches use Store-and-Forward by default.

L2 Limits & Exam Rules

Layer 2 Cannot Route
Switches have no concept of IP subnets. They cannot forward traffic between different networks without a router or Layer 3 switch.
Layer 2 Limitations
  • Cannot break broadcast domains (need VLANs or router)
  • Broadcasts flood entire VLAN — all hosts receive them
  • No routing between subnets without Layer 3
  • MAC table full → switch degrades to hub behavior (floods)
Lowest Bridge ID = Root Bridge
Default Priority = 32768
STP Convergence = 20+15+15 = ~50 sec
MAC age-out = 300 seconds
Each port = 1 collision domain
All ports (no VLAN) = 1 broadcast domain
