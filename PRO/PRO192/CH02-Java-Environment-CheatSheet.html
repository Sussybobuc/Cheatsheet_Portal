Classpath & Packages

CLASSPATH = list of directories and JARs where the JVM looks for classes
Setting Classpath
# Compile with classpath
javac -cp lib/mylib.jar src/Main.java

# Run with classpath (. = current dir)
java -cp .:lib/mylib.jar Main

# Windows uses ; separator
java -cp .;lib\mylib.jar Main
Package Naming
// File: src/com/company/Hello.java
package com.company;

public class Hello { ... }
💡 Convention
Package name = reverse domain (com.company.project). Maps directly to directory structure.
Import Statements
import java.util.ArrayList;     // single class import
import java.util.*;             // entire package (avoid in large projects)
import static java.lang.Math.*; // static import (use Math methods unqualified)
ℹ Java.lang
java.lang is auto-imported — no import needed for String, Math, System, Object, etc.

JShell — Interactive REPL

Introduced in Java 9. Test code snippets without writing a full class or main method.

$ jshell
|  Welcome to JShell -- Version 17
jshell> 2 + 3
$1 ==> 5
jshell> String s = "Hello"
s ==> "Hello"
jshell> s.toUpperCase()
$3 ==> "HELLO"
jshell> int[] arr = {1, 2, 3}
arr ==> int[3] { 1, 2, 3 }
jshell> for (int x : arr) System.out.println(x)
1
2
3
jshell> /exit

Build Tools Overview

ToolConfig FileKey Feature
Mavenpom.xmlConvention over config; Central Repository; standardized lifecycle phases
Gradlebuild.gradleFlexible DSL (Groovy/Kotlin); incremental builds; Android default

Maven Lifecycle Phases (in order)
  1. validate — validate project structure is correct
  2. compile — compile source code
  3. test — run unit tests
  4. package — create JAR/WAR artifact
  5. verify — run integration tests
  6. install — install to local repository (~/.m2)
  7. deploy — publish to remote repository
Maven pom.xml — Dependency Skeleton
<dependencies>
  <dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.13</version>
    <scope>test</scope>
  </dependency>
</dependencies>
💡 Run Maven
mvn compile · mvn test · mvn package · mvn clean install

Java Program Structure

// 1. Package declaration (optional, must be first)
package com.example;

// 2. Import statements
import java.util.Scanner;

// 3. Class declaration (matches filename)
public class MyProgram {

    // 4. Class fields (instance or static)
    private String name;

    // 5. Constructor
    public MyProgram(String name) {
        this.name = name;
    }

    // 6. Methods
    public void greet() {
        System.out.println("Hello, " + name);
    }

    // 7. Main entry point (static)
    public static void main(String[] args) {
        MyProgram p = new MyProgram("World");
        p.greet();
    }
}
  • One public class per file; filename must match public class name
  • main must be public static void main(String[] args)
  • args array receives command-line arguments (index 0 = first arg)
  • Package declaration must be the very first non-comment line
  • Imports come after package, before class declaration
