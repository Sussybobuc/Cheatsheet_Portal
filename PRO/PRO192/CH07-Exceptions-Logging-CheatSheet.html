Exception Hierarchy

Throwable → Error (JVM errors, don't catch) | Exception (application errors, catch these)
Throwable
├── Error (don't catch!)
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── VirtualMachineError
└── Exception (catch/handle)
    ├── RuntimeException (Unchecked)
    │   ├── NullPointerException
    │   ├── ArrayIndexOutOfBoundsException
    │   ├── ClassCastException
    │   ├── ArithmeticException (/ by zero)
    │   ├── NumberFormatException
    │   └── IllegalArgumentException
    └── Checked Exceptions
        ├── IOException
        ├── FileNotFoundException
        ├── SQLException
        └── ParseException

try / catch / finally

try {
    // Code that may throw exceptions
    int result = 10 / divisor;
    String s = null;
    s.length();  // NullPointerException
} catch (ArithmeticException e) {
    System.err.println("Division error: " + e.getMessage());
} catch (NullPointerException e) {
    System.err.println("Null pointer: " + e.getMessage());
} catch (Exception e) {
    // Catch-all (less specific, must come AFTER specific)
    e.printStackTrace();
} finally {
    // ALWAYS runs — cleanup resources
    System.out.println("Finally block");
}

// Multi-catch (Java 7+)
try {
    ...
} catch (IOException | SQLException e) {
    System.err.println("I/O or DB error: " + e);
}
  • catch blocks checked in order — put specific first
  • finally runs even if exception not caught
  • finally runs even after return
  • if exception in finally — original exception lost

Checked vs Unchecked

CheckedUnchecked (Runtime)
ExtendsExceptionRuntimeException
Must handle?Yes (compile error if not)No
CauseExternal factors (file, network)Programming errors
ExamplesIOException, SQLExceptionNPE, AIOOBE, CCE
throws declaration
// Must declare checked exceptions in signature
public void readFile(String path) throws IOException {
    FileReader fr = new FileReader(path); // checked
    ...
}

// Or handle it:
public void readFile(String path) {
    try {
        FileReader fr = new FileReader(path);
    } catch (IOException e) {
        System.err.println("File error: " + e);
    }
}
Tip
Debate: James Gosling (Java creator) stands by checked exceptions; most modern languages (Kotlin, C#) dropped them. In practice: catch what you can handle, propagate the rest.

Throwing Exceptions

throw statement
public void setAge(int age) {
    if (age < 0 || age > 150) {
        throw new IllegalArgumentException(
            "Invalid age: " + age);
    }
    this.age = age;
}

public void processFile(String path) throws IOException {
    if (path == null) {
        throw new NullPointerException("Path cannot be null");
    }
    // ... file processing
}

// Re-throw (with cause)
try {
    lowLevelMethod();
} catch (LowLevelException e) {
    throw new HighLevelException("High level message", e);
}
Tip
Include the original exception as cause when re-throwing — preserves the full stack trace chain

Custom Exceptions

// Checked custom exception
public class InsufficientFundsException extends Exception {
    private double amount;

    public InsufficientFundsException(double amount) {
        super("Insufficient funds: need " + amount + " more");
        this.amount = amount;
    }

    public double getAmount() { return amount; }
}

// Unchecked custom exception
public class DatabaseException extends RuntimeException {
    public DatabaseException(String message) {
        super(message);
    }
    public DatabaseException(String message, Throwable cause) {
        super(message, cause);
    }
}

// Usage
public void withdraw(double amount)
        throws InsufficientFundsException {
    if (amount > balance) {
        throw new InsufficientFundsException(amount - balance);
    }
    balance -= amount;
}

try-with-resources

Auto-closes resources that implement AutoCloseable when block exits
// Old way (verbose):
FileReader fr = null;
try {
    fr = new FileReader("file.txt");
    // use fr
} finally {
    if (fr != null) fr.close();
}

// try-with-resources (Java 7+):
try (FileReader fr = new FileReader("file.txt");
     BufferedReader br = new BufferedReader(fr)) {
    String line = br.readLine();
    // fr and br auto-closed even if exception
}

// Multiple resources — closed in reverse order
try (var conn = DriverManager.getConnection(url);
     var stmt = conn.createStatement()) {
    var rs = stmt.executeQuery("SELECT ...");
}
Custom AutoCloseable
public class ManagedResource implements AutoCloseable {
    public ManagedResource() { System.out.println("Opened"); }
    @Override
    public void close() { System.out.println("Closed"); }
}
Tip
Always use try-with-resources for streams, connections, and any AutoCloseable resource

Assertions & Logging

Assertions
// assert condition;
// assert condition : message;

double sqrt(double x) {
    assert x >= 0 : "sqrt requires non-negative input";
    return Math.sqrt(x);
}

// Enable assertions at runtime:
// java -ea MyClass
// java -ea:com.example... MyClass  (specific package)
// Disabled by default in production!

Exception Best Practices

  • Catch specific exceptions (not just Exception)
  • Don't swallow exceptions with empty catch blocks
  • Include meaningful error messages
  • Use try-with-resources for resource cleanup
  • Wrap low-level exceptions with domain exceptions
  • Prefer unchecked exceptions for programming errors
  • Document exceptions with @throws Javadoc
Anti-patterns to avoid
// BAD: empty catch (swallows exception silently)
try { ... } catch (Exception e) {}

// BAD: catching too broad
try { ... } catch (Exception e) { e.printStackTrace(); }

// BAD: using exceptions for control flow
try {
    int val = Integer.parseInt(str);
} catch (NumberFormatException e) {
    // This is valid — parseInt has no "safe" version
}

// GOOD: use Optional or return code when no exception is expected

Stack Traces

// Reading a stack trace:
Exception in thread "main" java.lang.NullPointerException
    at com.example.MyClass.method(MyClass.java:42)
    at com.example.Main.main(Main.java:10)
//  ^class          ^method  ^file    ^line
Getting stack trace in code
try {
    riskyOperation();
} catch (Exception e) {
    // Get string version
    StringWriter sw = new StringWriter();
    e.printStackTrace(new PrintWriter(sw));
    String stackTrace = sw.toString();

    // Get cause
    Throwable cause = e.getCause();

    // Get message
    String msg = e.getMessage();

    // All stack frames
    StackTraceElement[] frames = e.getStackTrace();
}
