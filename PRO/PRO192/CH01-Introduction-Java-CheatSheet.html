Java Platform Overview

Java = language + JVM + standard library. Write code once, run on any platform with a JVM installed.

Three Editions
  • Java SE — Standard Edition; core language, collections, I/O, networking
  • Jakarta EE (formerly Java EE) — Enterprise Edition; servlets, JPA, CDI, web services
  • Java ME — Micro Edition; embedded and mobile-constrained devices
Key Features
  • Write Once Run Anywhere: .java → .class (bytecode) → runs on any JVM
  • Automatic garbage collection — no manual memory management
  • Strong static type safety — errors caught at compile time
  • Security sandbox — bytecode verified before execution

JVM Architecture

Compilation Pipeline
Source.java → [javac] → Source.class (bytecode) → [JVM/JIT] → Machine code
JVM Components
  • Class Loader — loads .class files into memory
  • Bytecode Verifier — checks code safety before execution
  • Interpreter / JIT — executes or compiles bytecode to native code
  • Garbage Collector — automatically frees unreferenced heap objects
  • Runtime — provides libraries, threading, I/O
JDK / JRE / JVM
ToolMeaning
JDKJava Dev Kit = JRE + compiler + debugger + javadoc + tools
JREJava Runtime Env = JVM + standard class libraries
JVMJava Virtual Machine — executes bytecode only
Memory Areas
  • Heap — objects and arrays live here (GC manages)
  • Stack — method frames, local variables, return addresses
  • Metaspace (formerly PermGen) — class metadata, method bytecode
  • Program Counter — tracks current instruction per thread

Common Misconceptions

❌ "Java is just JavaScript"
Completely different languages. JavaScript is a browser scripting language; Java compiles to bytecode and runs on the JVM.
❌ "Java is slow"
JIT (Just-In-Time) compilation makes modern Java approach native C++ speed for most workloads.
❌ "Java has pointers"
Java has no raw pointers. Object references are type-safe and cannot be manually dereferenced or manipulated arithmetically.
❌ "Java replaces C++"
Different use cases — Java prioritizes safety and portability; C++ prioritizes low-level control and maximum performance.
❌ "Java is always free"
OpenJDK is free. Oracle JDK has commercial licensing requirements for production use after Java 11.
❌ "Java is only for enterprise"
Used in Android (historically Dalvik/ART), backend APIs, desktop apps, embedded systems, and scientific computing.

OOP Principles Overview

The 4 Pillars of OOP
PillarDescription
Encapsulation Bundle data + behavior; hide implementation details; expose state only via methods (getters/setters)
Inheritance Subclass inherits superclass fields and methods; use extends keyword; promotes code reuse
Polymorphism Same method name, different behavior; runtime dispatch; achieved through method overriding
Abstraction Hide complexity; expose clean interface; implemented via abstract classes and interfaces
💡 Car Analogy
Encapsulation = internals hidden under the hood
Inheritance = ElectricCar extends Car
Polymorphism = drive() behaves differently for each car type
Abstraction = you press the gas pedal, not the engine internals

Quick Start: Hello World

// HelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
Compile & Run
javac HelloWorld.java   // creates HelloWorld.class
java HelloWorld         // runs the program
⚠ File Naming Rule
Class name MUST match filename exactly (case-sensitive). HelloWorld → HelloWorld.java
Key Rules
  • Every Java program starts at main(String[] args)
  • The class containing main must be public
  • Only one public class per file (filename matches that class)
  • Package name maps to directory structure
  • System.out.println prints to standard output with a newline
  • Statements end with ; — blocks use { }
