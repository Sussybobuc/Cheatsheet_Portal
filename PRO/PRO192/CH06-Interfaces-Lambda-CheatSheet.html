Interfaces

Interface = contract of methods a class must implement. No state (mostly). Multiple interfaces allowed.
public interface Drawable {
    // Abstract method (implicitly public abstract)
    void draw();

    // Default method (Java 8+) — has implementation
    default void drawWithBorder() {
        System.out.println("--- border ---");
        draw();
        System.out.println("--- border ---");
    }

    // Static method (Java 8+)
    static Drawable noOp() { return () -> {}; }

    // Constant (implicitly public static final)
    int MAX_SIZE = 1000;
}

public class Circle implements Drawable {
    @Override
    public void draw() {
        System.out.println("Drawing circle");
    }
}

Comparable & Comparator

Comparable (natural ordering — implement in the class)
public class Employee implements Comparable<Employee> {
    private String name;
    private double salary;

    @Override
    public int compareTo(Employee other) {
        // Negative: this < other
        // Zero:     this == other
        // Positive: this > other
        return Double.compare(this.salary, other.salary);
    }
}

// Now can sort:
Employee[] staff = { /* ... */ };
Arrays.sort(staff); // uses compareTo
Comparator (external comparison — no need to modify class)
// Sort by name:
Arrays.sort(staff, Comparator.comparing(Employee::getName));

// Sort by salary descending:
Arrays.sort(staff,
    Comparator.comparingDouble(Employee::getSalary).reversed());

// Chained: sort by dept, then by salary
Arrays.sort(staff,
    Comparator.comparing(Employee::getDept)
        .thenComparingDouble(Employee::getSalary));
Tip
Use Comparator.comparing() with method references — clean and readable

Lambda Expressions

Lambda Syntax
// (parameters) -> expression   or   (parameters) -> { statements; }

// No parameters
Runnable r = () -> System.out.println("Hello");

// One parameter (type inferred, parens optional)
Predicate<String> isEmpty = s -> s.isEmpty();
Predicate<String> isEmpty2 = (String s) -> s.isEmpty(); // explicit type

// Multiple parameters
Comparator<String> cmp = (a, b) -> a.length() - b.length();

// Block body (multiple statements)
Comparator<String> cmp2 = (a, b) -> {
    int diff = a.length() - b.length();
    return diff != 0 ? diff : a.compareTo(b);
};
Method References (shorthand for lambdas)
KindSyntaxLambda Equivalent
Static methodMath::absx -> Math.abs(x)
Instance (bound)str::toUpperCase() -> str.toUpperCase()
Instance (unbound)String::toUpperCases -> s.toUpperCase()
ConstructorArrayList::new() -> new ArrayList<>()
Lambda Examples in Practice
import java.util.*;
import java.util.function.*;

List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

// Sort with lambda
names.sort((a, b) -> a.compareTo(b));
// Same with method reference:
names.sort(String::compareTo);

// forEach with lambda
names.forEach(name -> System.out.println(name));
// Method reference version:
names.forEach(System.out::println);

// Predicate
Predicate<String> longName = s -> s.length() > 4;
names.stream()
     .filter(longName)
     .forEach(System.out::println);  // Alice, Charlie

// Function
Function<String, Integer> strLen = String::length;
names.stream()
     .map(strLen)
     .forEach(System.out::println);  // 5, 3, 7
Tip
Lambdas can capture effectively-final local variables from the enclosing scope
Warning
Lambda cannot modify captured local variables (they must be effectively final)

Inner Classes

Member Inner Class
public class Outer {
    private int x = 10;

    class Inner {
        void show() {
            System.out.println(x);  // can access outer's private!
        }
    }

    void demo() {
        Inner inner = new Inner();
        inner.show();
    }
}

// From outside:
Outer.Inner inner = new Outer().new Inner();
When to use inner classes
  • Need access to outer class's private members
  • Logically grouped with outer class
  • Iterator implementations (classic use case)
  • Event listeners (GUI)

Static Nested Classes

public class LinkedList<E> {
    private Node<E> head;

    // Static nested — no access to outer instance
    private static class Node<E> {
        E data;
        Node<E> next;
        Node(E data) { this.data = data; }
    }

    public void addFirst(E data) {
        head = new Node<>(data);
    }
}

Anonymous Classes & Functional Programming

Anonymous class (pre-lambda way)
// Old way: anonymous class
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Running");
    }
};

// Modern way: lambda
Runnable r2 = () -> System.out.println("Running");

// Anonymous Comparator (old)
Arrays.sort(names, new Comparator<String>() {
    @Override
    public int compare(String a, String b) {
        return a.length() - b.length();
    }
});

// Lambda (modern)
Arrays.sort(names, (a, b) -> a.length() - b.length());
// Method reference (cleanest)
Arrays.sort(names, Comparator.comparingInt(String::length));
Tip
Use anonymous classes when you need to store state; use lambdas for single-method interfaces without state

Interface Default & Static Methods

public interface Collection<E> {
    boolean add(E element);     // abstract
    int size();                 // abstract

    // Default — backwards compatible addition
    default boolean isEmpty() {
        return size() == 0;
    }

    // Static utility
    static <E> Collection<E> empty() {
        return new EmptyCollection<>();
    }
}
Resolving default method conflicts
interface A { default void hello() { System.out.println("A"); } }
interface B { default void hello() { System.out.println("B"); } }

class C implements A, B {
    @Override
    public void hello() {
        A.super.hello();  // explicitly choose A's version
    }
}
Warning
If two interfaces have same default method, implementing class must override to resolve ambiguity

Predicate Composition & Function Chains

Predicate<Integer> isPositive = n -> n > 0;
Predicate<Integer> isEven = n -> n % 2 == 0;

// Compose predicates
Predicate<Integer> isPositiveAndEven = isPositive.and(isEven);
Predicate<Integer> isPositiveOrEven  = isPositive.or(isEven);
Predicate<Integer> isNegative        = isPositive.negate();

// Compose Functions
Function<Integer, Integer> times2 = x -> x * 2;
Function<Integer, Integer> plus3  = x -> x + 3;

// andThen: apply times2 then plus3
Function<Integer, Integer> times2Plus3 = times2.andThen(plus3);
times2Plus3.apply(5); // (5*2)+3 = 13

// compose: apply plus3 first then times2
Function<Integer, Integer> plus3Times2 = times2.compose(plus3);
plus3Times2.apply(5); // (5+3)*2 = 16
Tip
Function composition lets you build complex transformations from simple pieces — building blocks of functional style
