OOP Concepts Introduction

Object = State (fields) + Behavior (methods) + Identity (unique reference)
Key Terms
TermMeaning
ClassBlueprint/template for objects; defines structure and behavior
ObjectInstance of a class; created with new
FieldData stored in an object (instance variable)
MethodFunction defined in a class; operates on object's data
ConstructorSpecial method to initialize a new object
EncapsulationHiding data + behavior in one unit; access via methods
thisReference to the current object instance
Tip
Objects model real-world entities: BankAccount object has balance (field) and deposit() (method)

Defining a Class

public class Employee {
    // Instance fields
    private String name;
    private double salary;
    private LocalDate hireDate;

    // Constructor
    public Employee(String name, double salary) {
        this.name = name;
        this.salary = salary;
        this.hireDate = LocalDate.now();
    }

    // Getter (accessor)
    public String getName() { return name; }
    public double getSalary() { return salary; }

    // Setter (mutator)
    public void setSalary(double newSalary) {
        if (newSalary > 0) salary = newSalary;
    }

    // Method
    public void raiseSalary(double percent) {
        salary *= 1 + percent / 100;
    }
}
Usage
Employee emp = new Employee("Alice", 75000);
emp.raiseSalary(10);
System.out.println(emp.getSalary()); // 82500.0

Constructors

Rules
  • Same name as class, no return type
  • Called with new, can be overloaded
  • If no constructor defined → default no-arg constructor provided
  • Once you define any constructor → default gone
public class Circle {
    private double radius;

    // No-arg constructor
    public Circle() { this.radius = 1.0; }

    // Parameterized constructor
    public Circle(double radius) {
        if (radius < 0) throw new IllegalArgumentException();
        this.radius = radius;
    }

    // Constructor chaining with this()
    public Circle(int diameter) {
        this(diameter / 2.0);  // must be first statement
    }
}
Tip
Constructor chaining with this(...) avoids code duplication. Must be first statement.

Static Fields and Methods

static = belongs to CLASS, not instances. Shared by all objects.
public class MathUtils {
    // Static field (class variable)
    public static final double PI = 3.14159;
    private static int instanceCount = 0;

    public MathUtils() { instanceCount++; }

    // Static method (no 'this', no instance needed)
    public static double circleArea(double r) {
        return PI * r * r;
    }

    public static int getCount() { return instanceCount; }
}

// Usage: no object needed
double area = MathUtils.circleArea(5);
int count = MathUtils.getCount();
Warning
Static methods cannot access instance fields or call instance methods directly
Static initializer block
static {
    // runs once when class is loaded
    System.out.println("Class loaded");
}

Method Parameters

Java is ALWAYS pass-by-value. For objects, the reference (address) is copied.
Primitives vs Objects
void tripleValue(double x) {
    x = x * 3;  // only changes local copy
}

void tripleSalary(Employee emp) {
    emp.raiseSalary(200);  // modifies the object through reference
}

void swap(Employee a, Employee b) {
    // CANNOT swap references — only swaps local copies
    Employee temp = a; a = b; b = temp;  // no effect outside!
}
  • Primitive types → method gets a copy, cannot modify original
  • Object references → method gets copy of reference → can modify object's state
  • But cannot reassign the original reference
Tip
Think of object reference as a sticky note with an address. Method gets a copy of the note — can go to the house and change it, but can't move the original note.

Encapsulation & Design Patterns

Getters & Setters (JavaBeans pattern)
public class Person {
    private String name;   // private field
    private int age;

    // Getter: returns value
    public String getName() { return name; }

    // Setter: validates before setting
    public void setName(String name) {
        if (name == null || name.isEmpty())
            throw new IllegalArgumentException("Name cannot be empty");
        this.name = name;
    }

    public int getAge() { return age; }

    public void setAge(int age) {
        if (age < 0 || age > 150)
            throw new IllegalArgumentException("Invalid age");
        this.age = age;
    }
}
Tip
getName/setName convention = JavaBean property 'name'. Frameworks (Spring, Hibernate) rely on this.
Records (Java 16+ — immutable data classes)
// Compact, immutable data holder
public record Point(double x, double y) {
    // Auto-generated: constructor, getters, equals, hashCode, toString

    // Custom validation in compact constructor
    public Point {
        if (x < 0 || y < 0)
            throw new IllegalArgumentException();
    }

    // Can add methods
    public double distance(Point other) {
        return Math.sqrt(Math.pow(x - other.x, 2) +
                         Math.pow(y - other.y, 2));
    }
}

// Usage
Point p = new Point(3.0, 4.0);
System.out.println(p.x());      // accessor (not getX!)
System.out.println(p);          // Point[x=3.0, y=4.0]
Immutable class (pre-records way)
public final class ImmutablePoint {
    private final double x;
    private final double y;
    public ImmutablePoint(double x, double y) {
        this.x = x; this.y = y;
    }
    public double getX() { return x; }
    public double getY() { return y; }
    // no setters!
}
Tip
Immutable objects are thread-safe and easier to reason about. Prefer records for simple data carriers.

Packages & Imports

Package = namespace for classes. Maps to directory structure.
// Declare package (first line, before imports)
package com.company.project;

// Import single class
import java.util.ArrayList;

// Import all classes in package (avoid in large projects)
import java.util.*;

// Static import (import static members)
import static java.lang.Math.*;  // then use sqrt() not Math.sqrt()

// No import needed for java.lang (String, System, Math, etc.)
Naming Convention
  • Package: all lowercase, reverse domain (com.oracle.java)
  • Class: PascalCase (ClassName)
  • Method/field: camelCase (methodName)
  • Constant: UPPER_SNAKE_CASE (MAX_VALUE)

Object Class Methods

All classes extend Object implicitly

Key methods to override
public class Employee {
    private String name;
    private double salary;

    @Override
    public String toString() {
        return "Employee[name=" + name + ", salary=" + salary + "]";
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (!(obj instanceof Employee other)) return false;
        return Objects.equals(name, other.name)
            && Double.compare(salary, other.salary) == 0;
    }

    @Override
    public int hashCode() {
        return Objects.hash(name, salary);
    }
}
Warning
If you override equals(), you MUST override hashCode() too — contract requires equal objects have same hash
