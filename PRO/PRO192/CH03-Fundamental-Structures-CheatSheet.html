Use var only when the type is obvious from the right side:

Naming Rules
  • Must start with letter, $, or _ (not a digit)
  • Case-sensitive: myVar ≠ myvar
  • Convention: camelCase for variables, PascalCase for classes
  • Cannot use reserved words: int, class, if, etc.

Type Conversions

Widening (automatic, safe — no data loss)
byte → short → int → long → float → double
int i = 42;
long l = i;     // widening: automatic, safe
double d = l;   // widening: automatic
Narrowing (explicit cast — may lose data)
double d = 9.99;
int i = (int) d;     // i = 9  (truncated, NOT rounded!)
long big = 123456789012L;
int n = (int) big;   // may overflow!
⚠ Truncation, Not Rounding
(int)9.9 gives 9, not 10. Cast cuts the decimal part, it does not round.
Numeric Promotion in Expressions
  • byte/short/char auto-promoted to int in expressions
  • If either operand is long → result is long
  • If either operand is float → result is float
  • If either operand is double → result is double
Wrapper Conversion Methods
int i    = Integer.parseInt("42");
double d = Double.parseDouble("3.14");
String s = Integer.toString(42);
String h = Integer.toHexString(255);  // "ff"
int max  = Integer.MAX_VALUE;         // 2147483647
int min  = Integer.MIN_VALUE;         // -2147483648

  • () — parentheses (always first)

  • = += -= etc. — assignment (lowest)

Switch Statement & Expression

Traditional Switch (Statement)
int day = 3;
switch (day) {
    case 1: System.out.println("Mon"); break;
    case 2: System.out.println("Tue"); break;
    case 3: System.out.println("Wed"); break;
    default: System.out.println("Other");
}
⚠ Fall-Through Trap
Forgetting break causes fall-through — execution continues into the next case!
Switch Expression (Java 14+) — No Fall-Through
String result = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    case 4, 5 -> "Thurs or Fri";  // multiple labels
    default -> "Weekend";
};
Switch Expression with Block (yield)
String label = switch (score) {
    case 10 -> "Perfect";
    default -> {
        String grade = score >= 5 ? "Pass" : "Fail";
        yield grade + " (" + score + ")";  // yield returns value
    }
};
💡 Best Practice
Prefer switch expressions with -> arrows — no fall-through risk, no break needed, can return a value.

⚠ Strings Are IMMUTABLE
Every "modification" (toUpperCase, replace, trim, etc.) creates a new String object. The original is never changed.
String Equality — CRITICAL
String a = "hello";
String b = "hello";
String c = new String("hello");

a == b         // true  (same string pool reference)
a == c         // false (different objects in memory!)
a.equals(c)    // true  (compares content — ALWAYS use this)
a.equalsIgnoreCase(c)  // true (case-insensitive)
⚠ NEVER use == for String content!
Always use .equals() to compare String content. == compares object references, not values.
StringBuilder — Mutable String
StringBuilder sb = new StringBuilder();
sb.append("Hello");
sb.append(", ");
sb.append("World");
String result = sb.toString(); // "Hello, World"

// Method chaining:
String r = new StringBuilder()
    .append("a").append("b").append("c")
    .toString(); // "abc"

// Other useful methods:
sb.insert(5, "!");      // insert at index
sb.delete(2, 4);        // delete range
sb.reverse();           // reverse content
sb.length();            // current length
💡 StringBuilder vs String
Use StringBuilder in loops — string concatenation with + inside a loop creates a new String object each iteration → O(n²) performance!

Control Flow

if / else if / else
if (x > 0) {
    System.out.println("positive");
} else if (x < 0) {
    System.out.println("negative");
} else {
    System.out.println("zero");
}

// Ternary shorthand
String sign = x > 0 ? "positive" : (x < 0 ? "negative" : "zero");
Loops
// while — check condition first
int i = 0;
while (i < 5) { System.out.println(i); i++; }

// do-while — always executes at least once
do { i++; } while (i < 10);

// for — classic counted loop
for (int j = 0; j < 10; j++) {
    System.out.println(j);
}

// for-each (enhanced for) — iterate collections/arrays
int[] arr = {1, 2, 3};
for (int val : arr) {
    System.out.println(val);
}
Break, Continue & Labels
KeywordEffect
breakExit current loop or switch immediately
continueSkip rest of loop body; jump to next iteration
break labelExit a named outer loop
returnExit the current method (optionally with value)
// Labeled break to exit outer loop:
outer:
for (int i = 0; i < 5; i++) {
    for (int j = 0; j < 5; j++) {
        if (i == 2 && j == 3) break outer;
    }
}

Arrays

Declaration & Creation
int[] nums = new int[5];           // 5 zeros (default)
int[] nums2 = {1, 2, 3, 4, 5};    // array literal
String[] words = {"a", "b", "c"};  // String array

int[][] matrix = new int[3][4];    // 2D: 3 rows, 4 cols
int[][] jagged = {{1,2},{3,4,5},{6}}; // jagged (rows differ)
Accessing & Modifying
nums.length          // 5  (field, NOT a method!)
nums[0] = 10;        // set element at index 0
int val = nums[0];   // get element at index 0
// Valid indices: 0 to length-1
Arrays Utility (java.util.Arrays)
import java.util.Arrays;

// Copy
int[] copy  = Arrays.copyOf(nums, nums.length);
int[] range = Arrays.copyOfRange(nums, 1, 3); // [1,3)

// Sort (ascending, in-place)
Arrays.sort(nums);

// Binary search (array must be sorted first)
int idx = Arrays.binarySearch(nums, 3);

// Fill all elements with a value
Arrays.fill(nums, 0);

// Print arrays
System.out.println(Arrays.toString(nums));
System.out.println(Arrays.deepToString(matrix)); // 2D

// Compare
boolean equal = Arrays.equals(nums, copy);
⚠ ArrayIndexOutOfBoundsException
Thrown if index < 0 or index >= length. Always validate indices or use for-each.
💡 Dynamic Sizing
For resizable arrays, use ArrayList<Integer> instead. Arrays have fixed size once created.
